# Trajectory Shaping

`shape-work` shapes an evidence-aware path, not just artifact inventory.

## Minimum Questions

- what is the current objective
- what is the next executable frontier
- what dependencies actually matter now
- what proof will later count as completion
- where should the user be interrupted
- what discoveries should force re-shaping

## Work Types

- deterministic
- investigative
- taste-sensitive
- decision-heavy
- integration-heavy

## Typical Outputs

- one lightweight task
- one normal task contract
- implementation plan plus a small task graph
- richer plan/spec/task graph for larger work

## Anti-Pattern

Do not pretend all uncertainty can be decomposed away up front.
