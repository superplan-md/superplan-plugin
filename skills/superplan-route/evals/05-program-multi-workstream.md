# Eval: Program For Multi-Workstream Change

## Scenario

User request:

> "Redesign the onboarding flow across web app, backend lifecycle emails, and analytics instrumentation, and make sure the new funnel definitions stay consistent everywhere."

The work spans multiple interfaces, product truths, and workstreams.

## Expected Route

- `program`

## Why

- multiple surfaces must stay aligned
- richer artifact structure is needed before execution stays trustworthy
- graph-aware coordination matters

## Expected Artifact Pattern

- `plan.md`
- `tasks.md`
- `tasks/T-*.md`
- one or more specs if durable clarification is needed

## Fail If

- the skill chooses `slice` and loses necessary alignment structure
- the skill frames the work as a pile of independent task files
