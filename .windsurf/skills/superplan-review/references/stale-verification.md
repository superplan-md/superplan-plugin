# Stale Verification

Rerun when freshness matters to the acceptance claim.

## Usually Stale

- checks ran before material code changes
- browser verification used an older UI or branch
- contract meaning changed after verification

## Often Still Usable

- local edits do not affect the verified AC
- rerun would not change the proof basis

When in doubt, tie the rerun decision to material impact, not any change at all.
