# Writing Plans Evals

## Pressure Scenario

- approved design for multi-step work with dependencies, partial uncertainty, and a real verification path

## Pass Condition

The skill writes an execution-oriented change plan through the CLI, keeps plan/spec/task roles separate, avoids fake certainty or ceremonial decomposition, and makes the user-visible reasoning legible by leading with intent, surfacing real alternatives when they exist, and making a recommendation.
