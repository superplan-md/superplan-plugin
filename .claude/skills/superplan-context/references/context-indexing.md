# Context Indexing

Use `.superplan/context/README.md` as the entrypoint and `.superplan/context/INDEX.md` as the router.

## README Should Explain

- what context exists
- how the context layer is organized
- how it differs from active work artifacts

## INDEX Should Route

- architecture
- conventions
- workflows
- operational constraints
- other stable reference docs

Keep the context system navigable without loading everything.
